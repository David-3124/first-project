### Инструкиця по работе с Git(символом $ отображается начало командной строки, которая ждет запрос):
1. Создания папки Git-репозитрия, требуется перейти в пакпи которую желанием сделать репозиторием и использовать команды:
  - $ git init - данной командой создаем Git-репозитрий
2. Если требуется разгитить папку Git-репозитория используем команду:
  - $ rm -rf .git - данной командой удаляем папку .git
3. Для проверки статус репозитория, используем команду:
  - $ git status - проверка статуса репозитория
    Подготовка, добавление файлов и сохранение в репозиторий используя команды:
  - $ git add . - данной командой подготвлеиваем к добавлению всю текушую парку в репозиторий
  - $ git add tema.txt -данной командой подготвлеиваем к добавлению отдельный файл в репозиторий
  - $ git add --all - данной командой подготвлеиваем к добавлению все файлы в репозитории

  - $ git commit -m "Название коммита" - данной командой сохраняем изменения и называем внессеный изменения.

  - $ git push -u origin master - Данную команду используем только при первой загрузке в Git (если ошибка master заменить на main)
  - $ git push - вносим изменения в репозитории

### Основные команды при работе с командной строкой:
  - $ ls -выводит содержимое директории
  - $ cd projects/git -переход по папкам командой cd, в данном случаи переходим в папку git разположенную в projects
  - $ pwd -данной командой можно узнать в какой директории назодишься
  - $ cd ~ -переход в доманшуюю директорию
  - $ cd .. -перейти в папку на уровень выше
  - $ touch newFile.txt -создание нового файла newFile.txt с расширением
  - $ mkdir new_dir -создание новой директории new_dir
  - $ cp newFile.txt new_dir/ -копируем файл newFile.txt в папку new_dir
  - $ mv newFile.txt new_dir/ -перемещаем файл newFile.txt в папку new_dir
  - $ cat newFile.txt -для чтения текса файла newFile.txt
  - $ rm newFile.txt -удаление файлаn ewFile.txt
  - $ rmdir new_dir -удаление папки, если она пустая
  - $ rm -r new_dir -удаление папки со всем содержимым

### Хеширование коммитов
**Хеширование** (от англ. _hash_, «рубить», «крошить», «мешанина») — это способ преобразовать набор данных и получить их «отпечаток»
- Git преобразует информацию о коммитах с помощью алгоритма SHA-1 и для каждого из них рассчитывает уникальный идентификатор — хеш.
- Хеш — основной идентификатор коммита и позволяет узнать его автора, дату и содержимое закоммиченных файлов.
- Все хеши, а также таблицу соответствий `хеш → информация о коммите` Git хранит в папке `.git`.

### Логи 
 - $ git log -выводит список коммитов
   - строка из цифр и латинских букв после слова **commit** — это хеш коммита;
   - **Author** — имя автора и его электронная почта;
   - **Date** — дата и время создания коммита;
   - в конце находится сообщение коммита.
 - $ git log --oneline -получаем сокращенный лог. Появятся только первые несколько символов хеша каждого коммита и их комментарии.
